# Contributing

## Workflow

1. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** — Edit pages, components, or styles as needed.

3. **Test locally:**
   ```bash
   npm run dev
   ```

4. **Commit with clear messages:**
   ```bash
   git commit -m "Add/Fix/Update: brief description"
   ```
   Examples: "Add publications page", "Fix header alignment", "Update team bio"

5. **Push and create a pull request:**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Merge to `main`** — Once reviewed, your changes will auto-deploy to production.

## Branch Naming

- `feature/` — new pages or features (e.g., `feature/resources-page`)
- `fix/` — bug fixes (e.g., `fix/mobile-layout`)
- `docs/` — documentation updates

## Code Style

Follow the existing code patterns in the project — no strict linting rules, just keep it consistent with what's already there.
